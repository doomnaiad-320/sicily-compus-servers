#校园服务微信小程序

## 基础需求(必须遵守)
1. 一个毕业设计,所以整体架构围绕需求走,UI样式等不要太复杂.
2. 使用 https://github.com/arhamkhnz/next-shadcn-admin-dashboard 这个库作为后端UI库
3. 前端根据微信小程序模板自带的uikit.不要自己创作样式.
4. 调试使用 SQLite 正式环境使用mysql
## 项目说明
- 校园服务需要两个小程序端,用户端和服务者端.但是我们可以通过路由来分开.判断当前用户是user或者是service user 展示对应的用户界面样式
- 如果有用户申请兼职者,需要提交身份证信息,手机号验证,
- 后端需要审核兼职者是否批准入驻
- 用户发单,兼职者接单后处于已接单状态,不再展示.
### 需求:
- 用户端:
    1. 注册,登陆(微信小程序登录)
    2. 收获/服务地址管理
    3. 模拟支付
    4. 对话/聊天
    5. 发布服务类行订单.
    6. 订单管理(编辑,删除)
    7. 申请兼职者售后.
    8. 向平台申诉.
- 兼职端:
    1. 注册/登陆(微信小程序登录)
    2. 开始接单/暂停接单逻辑. 开始接单则可以抢单,暂停后只能看无法接单.
    3. 认证(提交身份证信息,验证手机短信,) 提交后端进行认证
    4. 每次只能接一单,只有当前订单完成才能继续接单.
    5. 个人收入图表.日志.工作时长。
    6. 余额，提现到支付宝。后端审核后点击已打款，扣除提现者对应金额。前端余额更新。
    7. 个人信息设置：提现设置，手动输入支付宝。
    8. 
- 管理后端：
    1. 用户管理（增删改查）
    2. 兼职者管理（增删改查）审核：申请中，已通过，已拒绝，兼职者个人信息：收款支付宝账号，身份信息（模糊脱敏处理）余额，接单量，好评率，差评率。工作时长等信息。
    3. 订单管理
    4. 评价管理。
    5. 提现管理
    6. 数据图表。
## 逻辑：
- 订单逻辑：
    * 订单流程：
    - 用户发布订单，兼职者在审核通过且接单状态时 点击咨询，与用户在对话页进行沟通，确认无误后。用户在对话页 发布请接单button。兼职者接单。开始服务。订单变成服务中并不再展示在订单页中，兼职者状态变更为服务中，并且不能再接单。服务完成后，用户在对话页或者订单卡片上点击已完成。等待用户确认。用户在已发布的订单中点击已完成。填写评价。订单完成。用户认为服务质量不好，则点击售后button，填写原因。兼职者继续服务，服务完成后继续已完成的流程。在服务过程中，用户可以进行申诉/取消申诉。申诉都需要填写理由/原因
- 兼职者入住流程：
    - 在用户界面-我的-申请入住服务。提交身份证信息（因为是毕业设计所以只要判断png格式即可。）手机号短信认证（模拟认证，只判断手机号位数，验证统一使用1111）手动输入技能。提交入住申请。后端接收到入住申请，开始进行审核，1.审核通过：审核订单从审核中变成已通过。2 审核未通过：填写未通过的原因，订单变为未通过。兼职者收到未通过信息。进行修改。再次提交。直到审核通过。
